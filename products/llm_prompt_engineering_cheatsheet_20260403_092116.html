LLM Prompt Engineering Cheatsheet

Self-contained HTML reference card covering chain-of-thought, few-shot, role prompting, structured output, and tool-use patterns with before/after examples for GPT-4o, Claude, and Gemini.

Chain-of-Thought

  • Add: "Think step by step before answering"
  • Add: "Show your reasoning in numbered steps"
  • Zero-shot CoT: just append "Let's think step by step"
  • Few-shot CoT: include worked examples with reasoning
  • Use for: math, logic, multi-step reasoning

Role Prompting

  • "You are a senior software engineer with 10 years of Python experience"
  • Add domain constraints: "You only answer about security topics"
  • Tone control: "Respond concisely, no filler phrases"
  • Persona lock: put role in SYSTEM, task in USER
  • Avoid: vague roles like "helpful assistant"

Structured Output

  • "Respond with a JSON object matching this schema: {...}"
  • "Output ONLY the JSON. No prose before or after."
  • Use Claude/GPT response_format={type:"json_object"}
  • Provide a one-shot example of the target JSON
  • Validate with Pydantic: model_validate_json(response)

Few-Shot Examples

  • Input: [example] -> Output: [example] (3-5 pairs)
  • Order matters: put most relevant examples last
  • Format: Q: ... A: ... for simple classification
  • Use diverse examples to prevent shortcut learning
  • Balance positive/negative classes in examples

Tool Use / Function Calling

  • Define tools with strict JSON Schema (avoid additionalProperties)
  • Use required[] to mark mandatory parameters
  • Descriptions are part of the prompt -- be specific
  • "Use the search tool if you are unsure about recent events"
  • Handle parallel tool calls: gpt-4o/claude support multiple

Context Management

  • Put instructions at the TOP (primacy effect)
  • Repeat critical rules at the BOTTOM (recency effect)
  • Use XML tags to delimit sections: ...
  • Summarize long docs before including them
  • Cache-friendly: put stable content (instructions) before dynamic

Anti-Patterns to Avoid

  • "Be creative AND precise" -- contradictory constraints
  • Negative-only instructions: "Don't do X" -- add "Instead, do Y"
  • Ambiguous length: "short" -- use "under 200 words"
  • Sycophancy triggers: "Please" / "Thank you" in every turn
  • Prompt injection risk: never concatenate raw user input into system

Model-Specific Notes

  • GPT-4o: strong at structured output + vision
  • Claude 3.5+: best at long-context, instruction following
  • Gemini 2.0 Flash: fast + grounded search built-in
  • Llama 3.2: good for local/private, needs explicit format
  • All: temperature 0 for deterministic extraction tasks
